## The per-capability implementation pattern

The codebase already encodes a uniform pattern; each new capability repeats it.
Reuse the existing plumbing — do not invent new structure:

1. **Wire types** — already generated by `sl-msg-template` for every
   `message_template.msg` message (both directions). For CAPS items, hand-write
   the LLSD body codec in `sl-wire`.
2. **Command** — add a variant to the `Command` enum (`sl-proto/src/command.rs`)
   with a doc comment.
3. **Encode** — map `Command` to the wire message in
   `sl-proto/src/session/{conversions,methods,circuit}.rs` (the
   `AgentIsNowWearing` builder in `circuit.rs` is the canonical example).
4. **Event** — add the inbound reply/notification to the `Event` enum
   (`sl-proto/src/types/event.rs`) plus any new payload type under
   `sl-proto/src/types/`, and decode the reply in the dispatch path.
5. **Server side** — add a `ServerEvent` variant and/or a `SimSession` `send_*`
   encoder in `sl-proto/src/sim_session.rs` (mirror `send_avatar_names` /
   `send_region_handshake`) so each new type is not decode-only.
6. **Runtimes (parity)** — surface the command/event through **both**
   `sl-client-tokio/src/lib.rs` and `sl-client-bevy/src/lib.rs`.
7. **REPL** — register the command in `sl-repl/src/registry.rs` (1:1 with
   `Command`) with arg parsing and a `build` fn.
8. **Tests** — a client decode test, a server encode test, and a loopback
   round-trip test, following the existing per-feature test style.
9. **Book** — document in the relevant `book/src/content/*.md` chapter (or a new
   chapter plus a `SUMMARY.md` entry). Keep prose wrapped at 80 columns; run
   `mdbook build book`, `rumdl`, and `typos` clean.

Mark each tier OpenSim-testable or SL-only; SL-only CAPS items guard on the
capability being present in the seed caps and defer live verification to a
future aditi run (like `SL_REPL_ROAD_MAP.md`'s E3).
